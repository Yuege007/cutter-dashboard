# 物料库存总览当前状态

更新日期：2026-04-27

本文记录 `MaterialOverviewCard.vue` 目前已落地的 Full 模式重构内容、数据口径和后续注意点。原计划文档见 `docs/plans/2026-04-23-inout-replenishment-api.md`。

## 1. 数据来源

当前仍复用物料列表接口：

```text
cachedApi.material.getMaterials(page, 100, {})
/cutter/toolCabinetApi/materialGrid
```

前端分页补拉全部物料后，在卡片内完成聚合计算。当前没有新增物料总览专用聚合接口。

主要计算项：

- `materialCount`：物料种类数。
- `totalQuantity`：总库存量。
- `totalValue`：库存总价值。
- `categorySummary`：按物料类别聚合。
- `materialValueTop`：物料级库存价值 Top。
- `materialQuantityTop`：物料级库存数量 Top。
- `heatmapGroups`：类别价值集中度热力块。

数值展示口径：

- `formatNumber`：按绝对值判断是否缩写为 `万` / `亿`，保留负号。
- `formatCurrency`：按绝对值判断是否缩写为 `万` / `亿`，保留负号和 `¥`。
- 示例：`-194926157` 显示为 `-1949.3万`，`-9233702960.67` 显示为 `-¥92.3亿`。

类别字段优先级：

```text
categoryName || materialType || cutterType || consumableType || '未分类'
```

说明：`consumableType` 在真实数据中偏粗分类，例如工具/耗材，必须放在兜底位置，否则类别分布会被压缩成少数大类。

## 2. Full 模式结构

当前 Full 模式由两部分组成：

```text
metrics-header
main-panel
```

已移除：

- Full 顶部标题、副标题。
- 顶部翻页按钮和页码。
- 自动轮播进度条。
- 右侧规模摘要、miniHeatmap、重点类别。
- 原始物料表格和表格分页。

保留：

- 顶部三项指标：物料种类、总库存量、库存价值。
- 主面板页签：类别分布、价值 Top、库存 Top、价值集中度。

顶部指标恢复为旧版 DOM 风格：

```html
<div class="metrics-header">
  <div class="metric-card">
    <div class="metric-value text-blue-500">...</div>
    <div class="metric-label">物料种类</div>
  </div>
</div>
```

样式使用 `display: flex` + `justify-content: space-around`，不再给单个指标卡加独立卡片框。

当前顶部指标已做横向收缩：

- 外层和指标项设置 `min-width: 0`。
- 长数字使用省略处理。
- 大数字和小标签字号已下调一档，减少窄卡片下的横向压力。

## 3. 四个页签

### 类别分布

左侧显示类别列表：

- 类别名。
- 物料种类数。
- 类别库存价值。
- 按绝对类别价值归一化后的条形比例。

右侧已从“价值占比带”改为“类别摘要”，避免和左侧明细重复。当前摘要项：

- 最高价值类别。
- Top 3 价值占比。
- 类别总数，并提示库存最多的类别。

类别摘要当前采用稳定的一行摘要布局：

```text
标签        值
```

说明：

- 不再依赖拖拽过程中的高度阈值切换普通态/紧凑态。
- 辅助说明不再占用垂直空间，改为 `title` 提示。
- 目标是在卡片缩放过程中始终保留核心文字，避免高度不足时出现空白摘要框。

价值占比使用绝对价值口径：

```text
abs(category.value) / sum(abs(category.value))
```

原因：当前数据可能存在负库存/负价值，不能直接用净总值做分母，否则占比容易被抵消成 0% 或失真。

### 价值 Top

已从类别级榜单改为物料级榜单。

排序口径：

```text
material.totalValue desc
```

展示方式：

- 第 1 名使用冠军卡突出展示。
- 第 2 名以后使用紧凑排名列表。

### 库存 Top

已从类别级榜单改为物料级榜单。

排序口径：

```text
material.inventory desc
```

展示方式：

- 排名圆点。
- 物料名称。
- 类别和库存价值。
- 库存数量。
- 底部库存水位条。

水位条按当前可见数据中的最大库存数量归一化。

### 价值集中度

保留纯热力块视图，不增加结论条。

展示内容：

- 类别简称。
- 类别库存价值。
- 类别物料种类数。

当前渲染上限：

```text
heatmapGroups.slice(0, 64)
```

原因：完整渲染所有类别可能造成 DOM 过多；固定最多 64 个热力块可以保持性能稳定，卡片变大时通过布局自然露出更多内容。

## 4. 自适应与渲染策略

列表类页签使用同一套可见数量策略：

```text
visible rows + 8 buffer
max 100
```

适用范围：

- 类别分布。
- 价值 Top。
- 库存 Top。

价值集中度固定最多 64 个块，不随高度计算动态渲染数量。

横向自适应补充：

- Full 外层、主面板、顶部指标、页签和榜单行均补充 `min-width: 0`。
- 页签限制最大宽度，文字超出省略。
- `价值 Top` 的冠军金额列和榜单金额列改为可收缩列，避免长金额撑出卡片。
- 顶部总库存量、库存价值使用短格式后，进一步降低横向溢出概率。

## 5. 已知取舍

- 库存 Top 的排名圆点目前按整行居中处理；底部水位条会影响视觉重心，后续若继续优化，建议调整行结构，而不是继续用 margin 微调。
- 价值集中度在极端尺寸下可能出现底部半行被裁切；当前优先保证性能和缩放稳定。
- 类别摘要为了稳定适配缩放，当前只常显标签和值；辅助说明仅通过 hover title 查看。
- 当前仍依赖物料列表接口做前端聚合；如果物料量继续增大，后续可考虑后端提供总览聚合接口。
