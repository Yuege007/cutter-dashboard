# 库存预警卡片风险列表轮播优化计划

日期：2026-06-05

## 背景

「库存预警」卡片目前在 Compact 和 Full 状态下都偏向“固定列表 + 滚动条”的呈现方式。数据量变多后，用户需要手动滚动才能看完整内容；卡片拉大时，可展示空间没有被充分利用；卡片较小时，列表又容易显得拥挤。

下一步将它作为“风险列表型卡片”的样板，改成和「刀柜货道状态」一致的自适应分页与自动轮播思路。

## 当前现状

- Mini：只显示预警总数，适合保留。
- Compact：显示几个统计数字，再展示少量预警列表，目前仍有滚动列表倾向。
- Full：有「全部 / 物料 / 货道」筛选，但列表区域主要依赖固定条数和滚动条。
- 当前列表条数由固定 slice 控制，卡片高度变化后不会自动增加或减少每页展示数量。

## 改造目标

- 去掉主要内容区对滚动条的依赖，改为自动分页。
- 卡片尺寸变大时，每页自动显示更多预警项；尺寸变小时，每页自动减少，避免文字裁切。
- Compact 状态改为摘要轮播，突出“有没有风险、最急的是谁”。
- Full 状态保留筛选能力，但列表变成动态分页 + 自动轮播。
- 手动切换筛选、上一页、下一页后，自动轮播重新计时，避免用户刚操作完就跳页。

## Compact 方案

Compact 不再尝试塞入完整列表，而是展示更适合小卡片的信息：

- 顶部保留核心数字：预警总数、物料预警、货道预警。
- 中部使用摘要轮播，按页展示：
  - 最高风险预警。
  - 优先处理物料。
  - 货道预警摘要。
- 每页只放 1-2 条重点信息，保证文字完整、层级清楚。
- 底部可加入轻量页码或进度条，提示当前轮播位置。

## Full 方案

Full 继续保留「全部 / 物料 / 货道」筛选，但把滚动表格改为分页风险列表：

- 根据列表容器高度动态计算每页可显示行数。
- 每条预警使用稳定高度的风险行，避免拖拽或刷新时布局跳动。
- 自动轮播当前筛选下的分页内容。
- 当前筛选页轮播完后继续循环当前筛选，不自动切换筛选，避免用户理解成本过高。
- 保留上一页 / 下一页按钮，用户可以手动查看。

## 风险行信息结构

每条预警建议包含：

- 风险类型：物料预警 / 货道预警。
- 物料名称或货道名称。
- 品牌、规格等辅助信息。
- 当前库存与预警阈值。
- 状态标签：缺货、低库存、需补充等。

文字较长时使用两行以内展示，超过部分省略，不撑高单行卡片。

## 实施步骤

1. 梳理 `InventoryAlarmCard.vue` 当前数据结构，确认物料预警和货道预警字段。
2. 增加列表容器尺寸监听，根据高度计算 `pageSize`。
3. 将固定 `slice` 的展示逻辑改为 `pagedWarnings`。
4. 增加自动轮播计时器、页码状态和手动切换重置逻辑。
5. 改造 Compact 为摘要轮播。
6. 改造 Full 列表样式，移除主要列表滚动条。
7. 补齐空状态和筛选切换状态。
8. 执行 `npm run build` 验证。

## 验收标准

- Compact 状态不出现列表滚动条。
- Full 状态主要预警列表不依赖滚动条。
- 拉高卡片后，每页预警数量自动增加。
- 缩小卡片后，文字不被上下裁切。
- 筛选切换、手动翻页、自动轮播互不冲突。
- 无预警、仅物料预警、仅货道预警三种情况都有正常展示。
- `npm run build` 通过。

## 边界说明

本阶段只优化「库存预警」卡片的信息呈现方式，不调整接口字段和预警规则。预警数据来源、阈值判断和业务含义保持不变。
