数字看板卡片 - 完整设计方案 (V4)
本文档是工具柜数字看板的最终设计指导，整合了内容规划与视觉设计规范。方案已融合 PatternFly 设计系统原则并针对中文信息展示进行深度优化，旨在为设计和开发团队提供一份清晰、统一、可执行的方案。
Ⅰ. 全局设计原则
1. 布局与色彩
布局网格 (Grid): 建议采用基于 12px 的网格系统。所有元素的尺寸、内外边距都应是 12px 的倍数，以建立视觉秩序感。
主色调: 采用深邃的蓝灰色或炭黑色作为看板背景，营造科技感和专业感。
色彩规范 (Color Palette):
数据色 (Data): 使用高亮的蓝色 (#0066CC)、青色 (#009596)、绿色 (#3E8635) 作为图表和关键数据的颜色。
状态色 (Status):
成功/在线/正常: 绿色 (#3E8635)
警告/预警: 橙色 (#F0AB00)
危险/错误/离线: 红色 (#C9190B)
进行中/信息: 蓝色 (#0066CC)
文本色 (Text): 主要文本使用白色或亮灰色 (#FFFFFF, #D2D2D2)，次要/说明性文本使用较暗的灰色 (#8A8D90)，形成视觉层级。
2. 中文字体设计规范 (Typography for Chinese)
中文字体结构复杂，需特别关注以保证可读性。
字体选择: 首选无衬线字体。推荐使用 PingFang SC (macOS/iOS), Microsoft YaHei (Windows), Source Han Sans (思源黑体) 作为跨平台解决方案。
字重 (Font Weight):
常规内容: 使用 Regular (常规) 字重。避免使用 Light 或 Thin，在深色背景下易模糊。
标题/强调: 使用 Medium 或 Semibold 字重。Bold (加粗) 在小字号下可能导致笔画粘连，需谨慎使用。
字号层级 (Font Size Scale):
核心指标 (KPI): 48px 或更大 (用于 Mini 卡片)
页面/大标题: 24px
卡片标题 (Full): 20px
卡片标题 (Compact): 18px
正文/表头: 14px (最常用的基础字号)
辅助/标签文字: 12px (不建议使用更小的字号)
行高 (Line Height): 为保证阅读舒适性，所有文本行高建议设置为字号的 1.5 至 1.7 倍。
3. 卡片状态 (Card States)
为提升用户体验，每张卡片都应具备以下状态：
加载中 (Loading): 卡片内容区域显示一个加载动画（如 Spinner），告知用户数据正在请求。
空状态 (Empty): 当无数据显示时，在卡片中央显示一个图标和提示文本，如“暂无数据”。
错误状态 (Error): 当数据加载失败时，显示错误图标和简短的错误信息，并提供“重试”按钮。
4. 内容溢出处理与滚动条规范
处理原则: 对于卡片内数据超出显示范围的情况（如长表格、多条目列表），必须使用内部滚动条进行处理，严禁使用自动轮播。此举旨在保障用户的控制权、信息的可发现性和整体可访问性。
滚动条设计规范:
样式: 滚动条应设计得纤细、低调，不干扰内容。
宽度: 8px。
轨道 (Track): 与卡片背景融合，使用半透明深色 (rgba(255, 255, 255, 0.05))。
滑块 (Thumb): 默认半透明灰色 (rgba(138, 141, 144, 0.4))，鼠标悬浮时加深颜色 (rgba(138, 141, 144, 0.7)) 以提供反馈。
圆角: 与卡片圆角保持一致 (4px)。

## 卡片类型概览
本系统包含以下8种核心卡片类型，每种卡片都支持Mini/Compact/Full三种视图模式：

1. **出入库趋势卡片** - 监控物料流动情况和趋势分析
2. **领用排行榜卡片** - 展示领用活跃度和排行信息
3. **库存状态概览卡片** - 提供库存健康度和预警信息
4. **智能柜状态监控卡片** - 监控硬件设备运行状态
5. **物料归还追踪卡片** - 追踪物料归还情况
6. **项目状态看板** - 项目整体状态概览
7. **物料库存总览卡片** - 物料库存全局概览和统计 ✨ **新增**
8. **用户状态统计卡片** - 系统用户基本情况统计

Ⅱ. 卡片设计详解
1. 出入库趋势卡片
业务目标: 实时监控物料流动情况，洞察出入库趋势。
🔹 Mini 视图 (1x1 ~ 2x2)
内容: 今日实时出入库总数。
数据源: API 2.3, 2.5 | 字段: number, payNum, payTime
视觉设计:
布局: 左右或上下结构。每个区域内垂直居中。
字体: 数字采用 48px+ Medium，标签（如“今日入库”）采用 14px Regular 灰色。
图标: 可在数字旁点缀简洁的“入”和“出”图标。
视觉示例:
+--------------------------------+
|                                |
|      128        |      56      |
|     今日入库    |     今日出库   |
|                                |
+--------------------------------+



🔸 Compact 视图 (3x2 ~ 4x3)
内容: 最近7天出入库趋势折线图。
数据源: API 2.3, 2.5 | 字段: number, payNum, payTime
视觉设计:
头部: 标题“近7日出入库趋势” (18px Medium)，右上角可放置一个“...” (Kebab) 菜单，内含“刷新”操作。
组件: 折线图隐藏网格线，线条平滑。鼠标悬浮在数据点上时，显示带有详细信息的 Tooltip。
视觉示例:
+--------------------------------+
| 近7日出入库趋势             ... |
+--------------------------------+
|       / \                      |
|      /   \   /\                |  <-- 简洁的折线图
|  ---/-----\-/--\----           |
|                                |
+--------------------------------+



🔶 Full 视图 (≥4x3)
内容: 可选时间范围的完整趋势图 + 本月入库物料Top 5结构表。
数据源: API 2.3, 2.5, 2.6 | 字段: productName, number 等
视觉设计:
头部: 标题“出入库详细分析” (20px Medium)，右侧为时间范围筛选器和“...”操作菜单。
布局: 左侧为功能更全的趋势图，右侧为Top 5物料表格。如果表格内容过长，应在此区域内应用自定义滚动条。
组件: 图表支持缩放和详细的Tooltip。表格条目清晰，可按数量排序，鼠标悬浮行有背景高亮效果。
视觉示例:

7. 物料库存总览卡片
业务目标: 提供物料库存的全局概览，包括物料种类、总库存量和总库存价值等关键指标。
🔹 Mini 视图 (1x1 ~ 2x2)
内容: 可切换显示的单一核心指标（物料种类/库存价值/总库存量）。
数据源: API materialGrid | 字段: productName, inventory, price
视觉设计:
布局: 垂直居中，主要数字突出显示。
字体: 数字 48px+ Medium，标签 12px Regular。
色彩: 物料种类（蓝色）、库存价值（红色）、总库存量（绿色）。
交互: 左下角悬停显示切换按钮（⟲），点击循环切换显示内容。
视觉示例:
+--------------------------------+
|                                |
|            ¥54.2万             |  ← 红色突出显示
|            库存价值             |
|                                |
|  ⟲                            |  ← 悬停显示切换按钮
+--------------------------------+

🔸 Compact 视图 (3x2 ~ 4x3)
内容: 主次分明的三指标展示，总库存价值为主要指标。
数据源: API materialGrid | 字段: productName, inventory, price
视觉设计:
布局: 主指标居中上方，辅助指标水平排列下方。
字体: 主指标 48px Medium（红色），辅助指标 32px Medium（蓝色/绿色），标签 12px Regular。
层次: 总库存价值最醒目，物料种类和总库存量作为补充信息。
