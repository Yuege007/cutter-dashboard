## 风险记录

- PDF 字段与真实接口可能不完全一致，先通过 adapter 做字段兼容。
- 部分接口要求 token 作为参数，当前 HTTP 客户端也会带 token header，第一版同时支持 query/body token。
- 旧工具柜 API、旧缓存包装和旧卡片注册表已删除；后续如发现残留旧业务文案，按可见程度逐步收口。
- 真实接口存在字段缺失或命名不一致情况，例如货道库存记录可能缺少柜体编号，需在聚合层补齐上下文字段。
- 如果先做完整 UI 精修，再继续补卡片，容易因卡片形态变化导致返工；当前策略是先补齐卡片矩阵，同时保持基础视觉规范。
