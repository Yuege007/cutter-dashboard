# 库存预警 Full 视图重构计划

更新日期：2026-04-27

本文记录 `InventoryAlarmCard.vue` 的 Full 视图重构计划。预览稿见：

```text
previews/inventory-alarm-full-preview.html
```

## 1. 定位边界

库存预警卡片关注“物料维度”的库存不足，用于仓库、采购、备料决策。

它和货道预警监控的边界如下：

- 库存预警：主语是物料，回答“哪些物料整体库存不足”。
- 货道预警监控：主语是柜体 + 货道，回答“哪些柜体货道需要现场补柜”。

因此库存预警 Full 视图不展示柜体、货道号、柜体分布等信息。

## 2. 当前现状

当前 `InventoryAlarmCard.vue` Full 视图是：

```text
标题 + 类别筛选 + 表格清单
```

现有数据来源：

```text
cachedApi.material.getWarnMaterials(1, 100)
```

当前表格字段：

- 物料名称。
- 当前库存。
- 预警库存。
- 差值。
- 类别。

现有问题：

- Full 模式只是表格清单，和“预警优先处理”的任务不够贴合。
- 缺少顶部风险概览。
- 表格列宽固定，缩放适配较弱。
- 接口当前只取前 100 条，若预警数据超过 100 条，Full 视图也只能展示前 100 条。

## 3. 目标设计

采用预览稿方向：

```text
顶部标题 + 类别筛选
三项风险指标
左侧优先处理物料列表
右侧类别分布
```

顶部三项指标：

- 断供物料。
- 低于预警。
- 预警总数。

不再单独展示“严重不足”指标，避免风险层级过多。

主列表为“优先处理物料”，每条展示：

- 物料名称。
- 风险标签：断供 / 低于预警。
- 规格或类别辅助信息。
- 当前库存 / 预警库存。
- 差值。
- 库存水位条。

右侧仅保留“类别分布”，不展示风险摘要，避免和左侧列表重复。

左侧优先处理物料和右侧类别分布都需要支持自动切换，交互方式参考货道预警监控：

- 按当前卡片高度计算每页可展示条数。
- 数据超过一页时自动轮播下一页。
- 顶部显示 `第 current / total 组`。
- 支持上一组 / 下一组按钮。
- 鼠标悬停或用户手动切换时可重置轮播计时。

## 4. 数据口径

每条预警物料派生字段：

```text
diff = inventory - inventoryWarn
progressPercent = inventoryWarn > 0 ? inventory / inventoryWarn * 100 : 0
isZero = inventory <= 0
level = isZero ? 'zero' : 'warn'
```

排序规则：

```text
断供优先
progressPercent 升序
diff 升序
```

顶部指标计算：

- 断供物料：`inventory <= 0`。
- 低于预警：`inventory < inventoryWarn`。
- 预警总数：筛选后的预警物料总数。

类别分布：

```text
按 cutterType 聚合数量
```

若后续发现 `cutterType` 过粗或为空，再补充字段兜底策略。

## 5. 自适应策略

Full 视图使用 CSS Grid：

```text
grid-template-rows: auto auto minmax(0, 1fr)
```

内容区域：

```text
grid-template-columns: minmax(0, 1fr) minmax(180px, 0.42fr)
```

列表分页与自动切换：

- 左侧优先处理物料使用 `alarmPage`。
- 右侧类别分布使用 `categoryPage`。
- 两侧分别计算 `pagedAlarmRows` 和 `pagedCategoryDistribution`。
- 每页数量根据对应面板高度估算，最少 1 条。
- 轮播间隔可先使用 6.5 秒，和现有货道监控/物料总览节奏保持接近。
- 若数据不足一页，不显示分页按钮和页码。

当前最终适配策略：

- 使用 `ResizeObserver` 观察左右面板高度。
- 先扣除标题区、进度条、列表 padding、gap 和安全余量，得到可用高度。
- pageSize 使用双约束计算：
  - 不允许多到低于最小行高。
  - 不允许少到超过最大行高。
- 当前页行高根据可用高度反推，并通过 CSS 变量传入列表项，让当前页尽量填满底部空间。
- 这样可以同时避免：
  - 底部最后一条被裁切。
  - 为了安全而留下明显空白。
  - 卡片拉高后长时间不增加条数。

当前行高范围：

```text
优先处理物料：56px - 78px
类别分布：30px - 42px
```

窄卡片策略：

- 外层、主列表、侧栏均设置 `min-width: 0`。
- 文本超出使用省略。
- 侧栏在较窄尺寸可下沉，不直接隐藏类别分布。
- 列表条目按分页数量展示，不使用浏览器滚动条撑破卡片。

## 6. 实施步骤

1. 保留 mini 和 compact 视图不变。
2. 替换 Full 模板：
   - 删除表格。
   - 添加三项指标。
   - 添加优先处理物料列表。
   - 添加类别分布侧栏。
3. 新增计算属性：
   - `alarmRows`
   - `zeroAlarmCount`
   - `warnAlarmCount`
   - `categoryDistribution`
4. 新增分页与自动切换状态：
   - `alarmPageIndex`
   - `categoryPageIndex`
   - `alarmPageCount`
   - `categoryPageCount`
   - `visibleAlarmRows`
   - `visibleCategoryRows`
   - 自动轮播 timer
5. 调整风险颜色：
   - 断供：红色。
   - 低于预警：橙色。
   - 总数：蓝色。
6. 添加横向和纵向适配样式。
7. 执行构建验证：

```text
npm run build
```
