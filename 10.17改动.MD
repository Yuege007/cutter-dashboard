10.17 改动说明（详版）

变更总览
- 累计改动 11 个文件：核心组件 3 个（趋势/排名/归还卡），配置与文档 3 个（.gitignore/.env.example/README.md），其余为依赖锁与构建相关文件。
- 目标：使中视图（Compact）与 max 视图（Full）在时间周期上保持同步，同时统一标题文案，确保文案与数据源一致。

代码改动详解
- InOutTrendCard.vue（入出库趋势卡）
  - 模板（Compact 视图）：
    - 标题由静态文案改为动态绑定 `{{ compactTitle }}`，随 Full 的 `selectedTimeRange`（7/15/30）变化展示“最近X天趋势”。
    - 图表绑定改为 `:option="compactChartOption"`，数据源为 `compactTrendData`（7 天走 `weekTrendData`，15/30 天走 `fullTrendData`）。
    - 汇总展示改为使用 `compactInTotal` 与 `compactOutTotal`，与所选周期一致（7 天用 `weekInTotal/weekOutTotal`，15/30 天用 `fullInTotal/fullOutTotal`）。
  - 脚本：
    - 新增 `compactTitle`（computed）：根据 `selectedTimeRange` 返回“最近7/15/30天趋势”。
    - 新增 `compactTrendData`（computed）：`selectedTimeRange==='7'` 返回 `weekTrendData`，否则返回 `fullTrendData`。
    - 新增 `fullInTotal/fullOutTotal` 与 `compactInTotal/compactOutTotal`（computed）：与所选周期对应的入/出库汇总数。
    - 将固定的 `weekChartOption` 替换为自适应的 `compactChartOption`：
      - 颜色与样式从主题 `themeStore.isDarkMode` 派生，统一 tooltip、axisPointer、grid、axis、series 样式。
      - series 双折线（出库/入库）与数据集保持一致。
  - 订阅/数据管道：
    - 保留并明确周期订阅：`week-pickups`、`15days-pickups`、`30days-pickups`，当 Full 选择对应周期时触发 `recomputeFullFromStore`，7 天同时维护 `weekTrendData` 以支持 Compact。

- UserRankingCard.vue（领用排行榜）
  - 模板（Compact 视图）：
    - 标题由静态“最近7天领用 Top 5”改为 `{{ compactTitle }}`。
    - 右侧时间文案仍为 `{{ formatTimeRange(timeRange) }}`，与当前 `selectedTimeRange` 计算出的时间段一致。
  - 脚本：
    - 新增 `compactTitle`（computed）：`selectedTimeRange==='today'` 显示“今日领用 Top 5”，否则显示“最近7天领用 Top 5”。
    - 维持 `selectedTimeRange`（'today' | 'week'）作为单一真源；`timeRange` 通过 `getTimeRange(selectedTimeRange.value)` 计算。
    - 排行数据生成逻辑（`fetchRankingData` + 聚合 +排序 Top N）保持不变，确保仅同步标题与时间周期。
    - 订阅保持：`today-pickups` / `week-pickups` 在对应选择下触发 `recomputeRankingsFromStore`，避免跨周期覆盖。

- MaterialReturnCard.vue（归还记录卡）
  - 模板（Compact 视图）：
    - 标题由静态“最近归还记录”改为 `{{ compactTitle }}`。
    - 右侧时间文案仍为 `{{ formatTimeRange() }}`，返回“今日/本周”。
  - 脚本：
    - 新增 `compactTitle`（computed）：`selectedTimeRange==='today'` 显示“今日归还记录”，否则显示“本周归还记录”。
    - 保留“最近归还记录列表”的语义（非 Top 5 按量排序），维持当前展示逻辑。
    - 订阅保持：`today-returns` / `week-returns` 仅在对应周期更新列表，避免错误覆盖。

视图同步行为（单一真源）
- 趋势卡：Full 选择 7/15/30 天 → Compact 标题/数据/汇总同步映射对应管道。
- 排名卡：Full 选择 今日/最近7天 → Compact 标题动态为“今日/最近7天领用 Top 5”。
- 归还卡：Full 选择 今日/本周 → Compact 标题动态为“今日/本周归还记录”。

追加改动（类型优化与标题配置）
- 标题配置与持久化
  - Toolbar 支持标题双击编辑：回车确认、Esc 取消、失焦保存。
  - 通过 `v-model:title` 与 `Dashboard.vue` 的 `dashboardTitle` 双向绑定。
  - 标题初始化优先级：`localStorage` → 环境变量 `VITE_APP_TITLE` → 默认值“数字看板系统”。
  - 默认标题更新：`.env.example` 中设置 `VITE_APP_TITLE="数字看板系统"`，作为团队示例配置。
  - 使用说明：双击标题进入编辑；如清空后将回退至默认标题。

- 轮询服务与类型优化
  - 统一将类型 `NodeJS.Timeout` 替换为 `ReturnType<typeof setInterval>` / `ReturnType<typeof setTimeout>`，适配浏览器类型系统。
  - 原因：项目基于 DOM tsconfig（不包含 Node 类型）；在浏览器中 `setInterval` 返回 `number`，与 `NodeJS.Timeout` 不兼容。
  - 受影响文件：
    - `vue-dashboard/src/services/polling.ts`
    - `vue-dashboard/src/services/cache/MemoryCache.ts`
    - `vue-dashboard/src/components/debug/CacheDebugPanel.vue`
    - `vue-dashboard/src/components/cards/UserRankingCard.vue`
    - `vue-dashboard/src/services/cache/utils.ts`
    - `vue-dashboard/src/utils/index.ts`
  - 行为影响：仅类型层面优化，不改变运行时逻辑；`clearInterval/clearTimeout` 用法保持一致。

- 验证与说明
  - 开发预览：`npm run dev` → 访问 `http://localhost:3000/`，双击标题可编辑且刷新后持久化。
  - 静态扫描：全局确认无 `NodeJS.Timeout` 残留引用。
  - 类型检查工具提示：`vue-tsc` 在 Node v22 环境下存在已知兼容问题（非业务代码导致），可选择升级 `vue-tsc` 或使用 Node v20 以通过 `npm run type-check`。

- 重置与回退

  - 如需临时禁用标题编辑，可在 `Toolbar.vue` 移除输入框与编辑状态逻辑，保留只读标题展示。
