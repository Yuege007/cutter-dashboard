# 刀具柜大屏卡片系统重构方案

更新日期：2026-05-20

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将当前工具柜数字看板重构为只服务刀具柜业务的电视大屏卡片系统，同时保留自由拖拽、缩放、布局保存和卡片三态展示能力。

**Architecture:** 保留现有 Vue 看板底座，替换业务数据层、轮询层和卡片内容层。新增刀具柜 API 封装与数据适配层，让卡片只依赖统一前端模型，不直接消费 PDF 原始字段。

**Tech Stack:** Vue 3、Vite、Pinia、TypeScript、ECharts、vue-grid-layout-v3、现有缓存与轮询基础设施。

---

## 1. 背景

当前项目原本面向工具柜数字看板，接口集中在 `/cutter/toolCabinetApi/*`。新的目标是单独作为刀具柜看板使用，接口依据：

```text
docs/toolcabinet-api-20260320.md
```

新接口体系为 `/plat/cutterApi/*`，业务重点包括刀具库存、货道库存、借刀、还刀、违规、补货、领料单、操作日志、盘点、工序和工单。

因此本次不是“给旧卡片换接口”，而是以现有看板底座为基础，重构成刀具柜专用卡片系统。

## 2. 产品定位

本系统用于工厂电视大屏，具有两类目标：

- **可视性：** 远距离看起来像完整的数字化管理大屏，有大数字、趋势、预警、动态消息和明确状态色。
- **实用性：** 现场人员能快速知道哪些刀具紧缺、哪些货道要补、今日领还情况如何、是否存在违规或异常。

系统不做固定大屏页面，而是延续原有卡片应用的核心能力：

- 用户可以添加、删除、拖拽、缩放卡片。
- 用户可以保存和恢复布局。
- 卡片根据尺寸自动切换 Mini / Compact / Full 内容。
- 提供推荐默认模板，但不强制固定布局。

## 4. 目标架构

```text
/plat/cutterApi/* 原始接口
  -> src/services/cutterApi.ts
  -> src/adapters/cutterAdapter.ts
  -> src/types/cutter.ts
  -> Pinia dataStore / polling
  -> 刀具柜卡片组件
```

关键原则：

- 卡片不直接使用 `product_name`、`brand_name`、`item_no_alias` 等后端字段。
- API 层负责请求。
- Adapter 层负责字段清洗、结构归一、默认值兜底。
- 卡片层只负责展示统一模型。

## 5. 第一阶段卡片库

第一阶段只做最核心的 6 张卡片，保证大屏能用、能看、能讲。

| 卡片 | 主要接口 | 业务问题 |
| --- | --- | --- |
| 刀具库存总览 | `cutterApi/consumablesInventory` | 当前刀具有多少、价值多少、哪些类别占比高 |
| 库存预警 | `cutterApi/consumableInventoryEarlyWarning`、`cutterApi/searchItemWarn` | 哪些刀具或货道需要处理 |
| 刀柜货道状态 | `cutterApi/searchCabinetInfo`、`cutterApi/cargoInventory` | 柜内货道是否正常、哪里缺刀 |
| 领刀趋势 | `cutterApi/searchByBorrowTime`、`cutterApi/trackStock` | 今天和最近几天领刀走势如何 |
| 领刀排行 | `cutterApi/searchByBorrowTime`、`cutterApi/wechatPieChart` | 谁领得多、哪些刀具用得多 |
| 归还追踪 | `cutterApi/searchByReturnTime` | 今天归还情况、近期归还记录 |

## 6. 卡片尺寸设计

每张卡片必须按尺寸自适应内容密度。

### 6.1 刀具库存总览

Mini：

- 一个核心大数字，在“刀具种类 / 总库存 / 库存价值”之间轮换或按配置固定。

Compact：

- 总库存价值。
- 刀具种类。
- 总库存量。
- 低库存数量。

Full：

- 顶部 KPI：刀具种类、总库存、库存价值、预警数量。
- 中部图表：类别库存分布、价值排行。
- 底部表格：型号、规格、品牌、物料编码、库存、预警值、单价、库存价值。

### 6.2 库存预警

Mini：

- 预警总数。
- 状态色：红色为断供，黄色为低库存。

Compact：

- 断供数量。
- 低库存数量。
- 紧缺 Top 5。

Full：

- Tab 1：耗材库存预警。
- Tab 2：货道库存预警。
- 每条展示型号、规格、品牌、物料编码、当前库存、预警值、货道号、柜号。
- 支持按类别、风险等级筛选。

### 6.3 刀柜货道状态

Mini：

- 预警货道数。
- 空货道数。

Compact：

- 柜体摘要。
- 正常 / 预警 / 空货道数量。
- 预警货道 Top 列表。

Full：

- 柜子选择器。
- 货道网格。
- 每个货道展示货道别名、刀具名称、库存 / 容量。
- 颜色含义：正常、低库存、空货道、禁用、未绑定。
- 点击或悬停展示货道详情。

### 6.4 领刀趋势

Mini：

- 今日领刀数量。

Compact：

- 近 7 天领刀趋势小图。
- 今日领刀和昨日对比。

Full：

- 近 7 / 15 / 30 天切换。
- 领刀趋势。
- 库存变更趋势。
- 分类趋势或高频刀具趋势。

### 6.5 领刀排行

Mini：

- 当前第一名和领刀数量。

Compact：

- 人员 Top 5 条形图。

Full：

- 人员排行。
- 刀具排行。
- 类别排行。
- 支持按今日、本周、近 7 天切换。

### 6.6 归还追踪

Mini：

- 今日归还数量。

Compact：

- 最近 5 条归还记录。

Full：

- 今日 / 本周归还明细。
- 归还人、领用人、刀具、货道、领用时间、归还时间。
- 若接口支持未归还状态，增加未归还列表；否则后续扩展。

## 9. 视觉规范

电视大屏优先级高于普通后台表格。

基础要求：

- 深色背景。
- 蓝 / 青作为主视觉色。
- 红色仅用于断供、严重异常。
- 黄色用于预警。
- 绿色用于正常。
- 数字要大，文字要少。
- 表格只展示 Top 或关键记录，不堆长表。
- Full 模式允许复杂内容，Mini 和 Compact 必须一眼读懂。
- 支持全屏模式，隐藏编辑工具栏。

字体建议：

- Mini 核心数字不小于 36px。
- Compact 核心数字不小于 28px。
- Full 标题不小于 18px。
- 辅助文字控制在 12px 到 14px。

## 13. 风险与处理

### 13.1 PDF 字段和真实接口不一致

处理：

- 以实测接口为准。
- Adapter 层做字段兼容。
- 每个接口保留原始响应日志开关，便于排查。

### 13.2 部分接口时间范围有限制

处理：

- 今日、近 7 天用明细接口。
- 更长周期优先用统计接口。
- 不强行用明细接口拉超大时间范围。

### 13.3 在线状态字段缺失

处理：

- 不把在线 / 离线作为第一版核心指标。
- 先展示柜体、货道、库存和预警。
- 如果后续确认有设备状态接口，再补设备状态卡片。

### 13.4 大屏视觉和自定义布局冲突

处理：

- 默认布局提供视觉效果。
- 用户仍可自由调整。
- 卡片自身负责适应尺寸，而不是依赖固定页面区域。
