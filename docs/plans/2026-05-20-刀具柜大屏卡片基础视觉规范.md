# 刀具柜大屏卡片基础视觉规范

创建日期：2026-05-20

适用阶段：Phase 3「卡片矩阵补齐 + 基础视觉规范」

关联文档：

- `docs/plans/2026-05-20-刀具柜大屏卡片系统重构方案.md`
- `docs/progress/2026-05-20-刀具柜大屏重构开发进度.md`

## 1. 设计判断

当前三张核心卡片已经跑通真实数据，但视觉仍偏工程态：

- 图标多为字符占位，业务识别弱。
- 卡片内容以文字和数字为主，缺少设备状态、货道状态、预警等级等视觉表达。
- 各卡片布局方式不同，尚未形成统一的大屏卡片语言。

本阶段不做完整 UI 翻新，先建立一套「新增卡片必须遵守」的基础规范，保证后续卡片矩阵补齐时不会继续扩大风格差异。

## 2. 视觉方向

关键词：

- 工业现场。
- 高密度信息。
- 设备状态优先。
- 深色大屏。
- 克制科技感。
- 少装饰，多状态。

不走营销式 hero、渐变堆叠和装饰性视觉。卡片应像一套生产现场的设备监控系统：数字清楚，状态明确，异常一眼能看到。

## 3. 卡片结构规范

每张卡片统一分为四个区域，按尺寸决定显示密度：

```text
┌──────────────────────────────┐
│ Header: 标题 / 操作             │
├──────────────────────────────┤
│ KPI: 主指标 / 次指标 / 单位       │
├──────────────────────────────┤
│ Body: 图表 / 列表 / 网格 / 排行   │
├──────────────────────────────┤
│ Footer: 更新时间 / 行动提示       │
└──────────────────────────────┘
```

不是每张卡都必须有 Footer，但 Header、KPI、Body 要尽量统一。

## 4. 图标规范

### 4.1 原则

- 禁止继续使用 `!`、`■`、箭头字符等作为正式卡片图标。
- 图标必须表达业务对象，而不是抽象装饰。
- 图标优先用于侧边栏组件库、空状态、图例和内容区识别，不默认进入卡片 Header。
- 卡片 Header 保持克制，避免图标和状态标签挤占业务数据注意力。

### 4.2 第一批业务图标

| 业务 | 图标语义 | 用途 |
| --- | --- | --- |
| 刀具库存 | 刀具 / 工具箱 / 物料层叠 | 刀具库存总览、分类分布 |
| 库存预警 | 警示三角 / 脉冲告警 | 库存预警、异常记录 |
| 刀柜货道 | 柜体 / 货道格栅 | 刀柜货道状态 |
| 领刀 | 向外箭头 / 领取记录 | 领刀趋势、领刀排行 |
| 归还 | 向内箭头 / 回收 | 归还追踪 |
| 库存变更 | 流向 / 交换箭头 | 库存变更记录 |
| 补货 | 加号 / 入库托盘 | 补货单状态 |
| 操作日志 | 文档 / 时间线 | 操作日志 |

### 4.3 实现建议

当前项目没有安装 lucide 图标库，已有 `src/components/icons` 目录。Phase 3 建议先新增本地 SVG 图标组件：

```text
src/components/icons/cutter/
  CutterToolIcon.vue
  CutterCabinetIcon.vue
  CutterWarningIcon.vue
  CutterBorrowIcon.vue
  CutterReturnIcon.vue
  CutterStockFlowIcon.vue
  CutterReplenishIcon.vue
  CutterLogIcon.vue
```

后续如果愿意增加依赖，可统一切换到 `lucide-vue-next`。

## 5. Header 规范

Header 统一包含：

- 左侧：卡片标题。
- 右侧：刷新、锁定、设置、删除等操作按钮。

样式建议：

- Header 高度：44px 左右。
- 标题字号：14px 到 16px。
- 操作按钮默认弱化，hover 时显示。

标题不要承担业务说明，说明应进入 Tooltip 或卡片 Body。实时、需处理、设备等状态信息不放 Header，确有必要时进入卡片内容区或 Footer。

## 6. KPI 规范

### 6.1 主数字

Mini 模式：

- 只保留 1 个主数字。
- 数字字号不小于 36px。
- 文案不超过 8 个汉字。

Compact 模式：

- 2 到 4 个 KPI。
- 主数字字号 24px 到 30px。
- 每个 KPI 必须有清楚单位或标签。

Full 模式：

- 顶部 3 到 5 个 KPI。
- KPI 下方接图表、列表或设备网格。

### 6.2 KPI 样式

KPI 块统一使用轻量背景，不再做厚重卡中卡。

建议视觉：

- 背景：`rgba(148, 163, 184, 0.06)`。
- 边框：`1px solid rgba(148, 163, 184, 0.16)`。
- 圆角：8px。
- 主数字用状态色或主题色。

## 7. 状态色规范

状态色必须稳定，不能每张卡自行解释。

| 状态 | 颜色变量 | 含义 |
| --- | --- | --- |
| 正常 | `--color-success` | 库存正常、货道正常、任务完成 |
| 预警 | `--color-warning` | 低库存、待处理、轻微异常 |
| 严重 | `--color-danger` | 断供、空货道、失败、严重异常 |
| 主信息 | `--color-primary` | 总量、当前选中、趋势主线 |
| 次信息 | `--color-text-secondary` | 单位、辅助说明、弱标签 |

红色只用于真正需要处理的异常，不能用作普通强调色。

## 8. Body 内容规范

### 8.1 列表类卡片

适用：预警、归还、操作日志、异常记录。

规则：

- Compact 最多显示 5 条。
- Full 最多默认显示 12 条，可滚动。
- 每条必须有左侧状态条或状态点。
- 每条优先展示「对象 + 状态 + 数量」，少展示长文本。

### 8.2 网格类卡片

适用：刀柜货道状态。

规则：

- 货道格子必须表达状态，不只是文本。
- 状态优先级：禁用 > 空货道 > 预警 > 正常。
- 格子内最多显示 3 层信息：货道号、库存数、刀具简称。
- Full 才展示完整网格；拖拽中可显示轻量摘要以保证性能。

### 8.3 图表类卡片

适用：趋势、排行、分类分布。

规则：

- 图表最多使用 2 个主色。
- 不使用复杂图例占用空间。
- 远距离可读优先，折线/柱状要比表格更优先。
- Full 模式允许图表 + Top 列表组合。

## 9. 三态内容规范

每张卡都必须明确 Mini / Compact / Full。

Mini：

- 1 个核心结果。
- 只回答「现在怎么样」。

Compact：

- 2 到 4 个指标。
- 1 个小列表或小图。
- 回答「哪里需要关注」。

Full：

- 完整操作视图。
- 包含筛选、分组、网格、趋势或明细。
- 回答「为什么、在哪里、下一步处理什么」。

## 10. 当前三张核心卡片的调整方向

### 10.1 刀具库存总览

保留：

- 刀具种类、总库存、库存价值、预警数量。
- 类别库存分布和价值排行。

优化：

- 侧边栏图标改为刀具/工具箱业务图标，卡片 Header 不展示图标。
- KPI 块统一为规范样式。
- Full 中减少表格感，增加库存分布条和 Top 价值条形表达。

### 10.2 库存预警

保留：

- 耗材预警和货道预警聚合。
- 断供 / 低库存区分。

优化：

- 侧边栏图标改为告警业务图标，卡片 Header 不展示图标。
- 每条预警增加风险状态条。
- 严重预警置顶，并使用更明确的行动文案，例如「需补货」「需排查货道」。

### 10.3 刀柜货道状态

保留：

- 刀柜选择器。
- 货道状态网格。
- 拖拽中轻量摘要。

优化：

- 侧边栏图标改为柜体/货道图标，卡片 Header 不展示图标。
- 左侧刀柜列表加入状态灯和数量分层。
- 右侧货道格子增强设备面板感，减少纯文字。

## 11. 新增卡片必须遵守的落地清单

新增每张卡时必须完成：

- [ ] 侧边栏使用业务 SVG 图标，不使用字符占位；卡片 Header 默认不展示图标。
- [ ] Mini / Compact / Full 三态都可用。
- [ ] Header 结构与现有核心卡片一致。
- [ ] KPI 使用统一密度和字号。
- [ ] 正常 / 预警 / 严重状态色含义一致。
- [ ] Compact 不超过 5 条列表。
- [ ] Full 不堆长表，优先图形化或 Top 列表。
- [ ] 拖拽时避免大列表、大网格、大图表持续重绘。
- [ ] `npm run build` 通过。

## 12. Phase 3 建议执行顺序

1. 新增业务图标组件目录。
2. 先把侧边栏注册项替换为业务图标。
3. 保持 `BaseCard` Header 为标题 + 操作按钮，不加入图标徽章和冗余状态标签。
4. 抽出通用 KPI / 内容区状态标签 / 风险行样式。
5. 按卡片矩阵继续补齐：
   - 领刀趋势。
   - 领刀排行。
   - 归还追踪。
   - 库存变更记录。
   - 分类 / 品牌 / 规格分布。
6. 等卡片矩阵完成约 80%，进入 Phase 4 完整 UI 视觉升级。

## 13. 暂不做的事

Phase 3 暂不做：

- 不重做整站背景和布局。
- 不引入复杂动画。
- 不做装饰性大屏 hero。
- 不为了视觉效果牺牲拖拽性能。
- 不把所有卡片都变成独立风格。

这些留到 Phase 4 统一处理。
