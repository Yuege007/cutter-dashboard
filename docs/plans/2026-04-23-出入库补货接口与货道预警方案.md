## 二、智能柜状态监控增加预警货道看板（第一版已落地）

### 背景

当前项目已有“库存预警”卡片，使用接口：

```text
/cutter/toolCabinetApi/findInventoryAlarmMaterial
```

该接口是物料汇总维度，适合回答“哪些物料库存不足”。

新版文档新增接口：

```text
/cutter/toolCabinetApi/channelInventoryAlarm
```

该接口是货道维度，适合回答“哪个柜子的哪个货道需要补货”。

当前项目也已有“智能柜状态监控”卡片，Full 模式使用：

```text
/cutter/toolCabinetApi/searchItemList
```

它按单个柜子展示所有货道，更适合柜体巡检和查看单柜全貌。新版 `channelInventoryAlarm` 不建议单独再做一张普通明细表卡片，避免和智能柜 Full 模式重复。

### 产品方案

在现有“智能柜状态监控”卡片 Full 模式中增加页签：

```text
柜体总览 | 预警货道
```

- `柜体总览`：保留现有逻辑，选择柜体后展示该柜所有货道。
- `预警货道`：新增大屏看板页，使用 `channelInventoryAlarm` 展示全公司预警货道。

### 呈现方式

预警货道页不使用滚动明细表，按车间电视/大屏场景设计。

顶部指标：

- 预警货道
- 断供货道
- 待补柜体
- 优先处理柜

中间区域：

- 展示“优先处理”货道卡片。
- 每页展示 3-5 条，具体数量按卡片尺寸自适应。
- 预警货道较多时自动轮播，不依赖人工滚动。
- 排序规则：
  - 断供货道优先。
  - 库存比例 `surplus / inventory` 越低越优先。
  - 同优先级下尽量按柜体聚合，方便现场一次处理。

右侧区域：

- 展示柜体预警分布。
- 按每个柜子的预警货道数排序。
- 用条形进度显示不同柜体的风险集中程度。

底部区域：

- 显示轮播页码，例如 `自动轮播 1 / 2`。
- 显示排序说明。

当前预览文件：

```text
previews/cabinet-channel-alarm-preview.html
```

### 数据接口

新增 API：

```http
POST /cutter/toolCabinetApi/channelInventoryAlarm
Content-Type: application/json;charset=UTF-8
token: <token>
```

计划默认请求：

```json
{
  "rwarnFlag": 0,
  "pageNum": 1,
  "pageSize": 100
}
```

说明：

- `rwarnFlag: 0` 查询库存不足货道。
- 后续如需要展示“正常货道”或完整统计，可再请求 `rwarnFlag: 1` 或增加筛选。
- 第一版建议只接库存不足货道，避免请求和 UI 复杂度过高。

## 三、预警货道看板 UI 优化计划

> 本节记录 2026-04-23 第一版 UI 优化方案与反馈问题；截至 2026-04-24，主要优化已落地，当前真实状态见第五节。

### 当前反馈问题

1. **卡片拉伸适配不足**

看板卡片可被用户随意调整大小，当前第一版在高度不足时仍强行展示固定数量的优先处理货道，导致物料名称、库存数字、状态标签出现挤压或溢出。

2. **轮播提示方式不够清晰**

当前底部有一排轮播点和说明文案：

```text
只轮播优先处理列表，顶部指标和柜体分布保持固定。
```

这些内容不适合车间电视看板，需要去掉。轮播信息应放在“优先处理”板块内部。

3. **轮播变化不明显**

当前主要是页码和列表内容变化，视觉反馈偏弱。用户远距离观看时，不容易感知优先处理列表已经切换。

4. **浅色主题表现较差**

深色主题整体可接受，但浅色主题下背景灰块过重，容易显得像禁用态。需要分别适配浅色/深色主题，而不是只按深色预览设计。

### 优化目标

- 不依赖滚动条。
- 不出现文字互相覆盖、跑出容器、压住库存数字的问题。
- 卡片高度变大时自动展示更多条，变小时自动减少条数。
- 只让“优先处理”板块轮播，顶部指标和柜体分布保持固定。
- 去掉底部说明文案和底部 dots。
- 轮播切换要有明显但不花哨的视觉反馈。

### 动态高度适配方案

使用 `ResizeObserver` 监听“优先处理”面板的实际高度，动态计算每页展示条数。

计算逻辑：

```text
可用高度 = 优先处理面板高度 - 标题栏高度 - 进度条高度 - 内边距
每条高度 = 任务卡固定高度 + gap
每页条数 = floor(可用高度 / 每条高度)
```

第一版条数范围：

```text
最少 2 条
最多 6 条
```

建议初始参数：

```ts
const rowHeight = 74
const reservedHeight = 58
const count = Math.floor((priorityPanelHeight.value - reservedHeight) / rowHeight)
return Math.max(2, Math.min(6, count))
```

2026-04-24 已调整为当前实现：

```ts
const reservedHeight = 60
const rowHeight = 58
const count = Math.floor((priorityPanelHeight.value - reservedHeight) / rowHeight)
return Math.max(1, count)
```

说明：

- 优先处理列表最少展示 `1` 条。
- 不再设置最大条数上限，卡片高度足够时尽量多展示。
- 从非 Full 模式回到 Full 模式时，会重新测量面板高度并重启轮播。

实现要点：

- 给优先处理面板增加 `ref`。
- 用 `ResizeObserver` 记录面板高度。
- 将当前固定 `alarmPageSize = 4` 改成 computed。
- `alarmPageCount` 和 `pagedPriorityChannels` 都使用动态 `alarmPageSize.value`。
- 当条数变化导致当前页超出范围时，将 `alarmPage` 归零或裁剪到最后一页。

### 优先处理任务卡布局约束

每条任务卡必须固定布局，不允许内容撑开卡片。

建议列：

```text
[货道号固定宽度] [物料信息自适应] [库存固定宽度] [状态固定宽度]
```

约束：

- 货道号固定宽度。
- 库存区域固定宽度，显示 `0 / 10`。
- 状态标签固定宽度，显示 `断供` 或 `紧缺`。
- 物料信息区域使用 `min-width: 0`。
- 物料名称、规格、柜体名称都必须 `overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`。
- 高度不足时隐藏低优先级辅助信息，例如规格或柜体编号，而不是让文字溢出。

### 轮播方式优化

移除：

- 底部轮播 dots。
- 底部说明文案。

改为在“优先处理”标题区域展示：

```text
优先处理                         第 13 / 20 组
[细进度条]
```

推荐动效：

- 细进度条 8 秒从 0 到 100%。
- 走满后切换下一组。
- 切换时“优先处理列表”整体淡入并轻微上移/横移。
- 顶部指标和柜体分布不动。

推荐第一版实现：

- 用 `alarmPage` 作为列表容器 `key`，触发 Vue transition。
- 使用 `<Transition name="alarm-page">` 包裹当前页列表。
- CSS 实现：

```css
.alarm-page-enter-active,
.alarm-page-leave-active {
  transition: opacity 0.28s ease, transform 0.28s ease;
}

.alarm-page-enter-from {
  opacity: 0;
  transform: translateY(6px);
}

.alarm-page-leave-to {
  opacity: 0;
  transform: translateY(-6px);
}
```

进度条：

```css
.alarm-progress-fill {
  animation: alarm-progress 8s linear infinite;
}
```

切换页码时重置 animation，保证进度条和轮播同步。

### 柜体分布优化

- 保留 `柜体分布 / 按预警数排序`。
- 去掉底部说明块。
- 按右侧面板高度动态计算每页条数。
- 最少展示 `1` 条，不设置最大条数上限。

## 六、2026-04-23 物料库存总览改版开发计划

- 表格区域依赖滚动和底部分页按钮浏览。

- 表格阅读成本高；
- 滚动方式不适合大屏；

将“物料库存总览”改成 **固定总览 + 内容切页** 的看板结构，优先回答：

1. 现在总共有多少种物料；
2. 总库存量是多少；
3. 总库存价值是多少；
4. 库存规模主要集中在哪些类别；


如果保留“聚合热力”页，热力图不负责承载完整详情。

要求：
