# 底部消息轮播自动消息计划

## 1. 背景

当前底部消息轮播由 `BottomBanner.vue` 内部维护：

- 组件内置默认消息。
- 用户可手动新增、删除、清空消息。
- 消息通过 `localStorage` 保存在本地。

现状适合人工公告，但不能自动反映看板中的库存、货道、出入库和领用变化。

## 2. 目标

在保留手动消息能力的基础上，增加“自动业务消息”。

自动消息不新增接口，第一版复用现有全局轮询数据：

- `warn-materials`
- `channel-inventory-alarms`
- `today-pickups`
- `today-replenishments`
- `materials-overview`
- `user-status`

底部轮播最终展示：

```text
自动业务消息 + 用户手动消息
```

系统运行状态、数据更新时间、无异常提示等系统类消息暂不纳入。

## 3. 数据获取方式

项目启动时，`main.ts` 会执行：

```ts
setupDefaultPollingTasks()
pollingService.start()
```

`pollingService` 会定时调用现有接口，并把结果写入 `useDataStore()`。

底部消息轮播不直接请求接口，只监听 `dataStore` 中已有数据：

```ts
dataStore.getData('warn-materials')
dataStore.getData('channel-inventory-alarms')
dataStore.getData('today-pickups')
dataStore.getData('today-replenishments')
dataStore.getData('materials-overview')
dataStore.getData('user-status')
```

这样可以避免重复请求，并保证底部消息与卡片数据一致。

## 5. 默认开启项

第一版建议默认开启以下业务消息：

1. 库存预警汇总
2. 断供物料提醒
3. 货道预警汇总
4. 今日出入库汇总
5. 今日领用冠军

其余预设消息放入“自动消息设置”中，由用户手动开启。

## 6. 更新策略

自动消息不维护独立请求周期，跟随现有轮询任务更新：

- 库存预警：约 1 分钟。
- 货道预警：约 1 分钟。
- 今日领用：约 45 秒。
- 今日补货：约 1 分钟。
- 物料总览：约 2 分钟。

当对应 `dataStore` 数据变化时，重新生成自动消息。

为避免轮播文字频繁跳动，可增加最短重算间隔，例如 30 秒。

## 7. 实现建议

建议新增自动消息生成逻辑，避免把业务聚合全部堆在 `BottomBanner.vue`。

可选文件：

```text
vue-dashboard/src/composables/useBottomBannerMessages.ts
```

职责：

- 读取 `useDataStore()`。
- 按预设开关生成自动消息。
- 合并自动消息和手动消息。
- 输出给 `BottomBanner.vue` 展示。

`BottomBanner.vue` 继续负责：

- 横向滚动展示。
- 暂停 / 播放。
- 手动消息管理。
- 自动消息开关管理入口。

## 8. 后续待定

- 自动消息开关是否需要持久化到 `localStorage`。
- 是否给自动消息增加优先级，例如断供消息优先展示。
- 是否区分消息类型颜色，例如库存预警、货道预警、出入库、排行。
- 是否允许用户隐藏全部自动消息，只保留手动消息。
