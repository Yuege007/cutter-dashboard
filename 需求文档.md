# 📘 Vue 版数字看板系统开发需求文档（2025全新版本）

## 🧭 一、项目定位与目标

构建一套基于 Vue 3 的数字看板系统，用于在工业/运营/仓储等场景中展示各类关键业务数据。该系统强调：

* 卡片化信息展示；
* 用户自由拖拽与缩放布局；
* 多尺寸下展示不同层级内容；
* Apple 风格视觉体验；
* 灵活接入后端 API 并保持数据更新。

---

## ⚒️ 二、核心功能模块

### 2.1 主看板（Dashboard）

* 基于 `vue-grid-layout` 实现卡片的拖拽与缩放；
* 支持 12 列网格系统，每张卡片定义 w/h/x/y；
* 卡片位置与尺寸变化实时同步状态管理；
* 拖拽/缩放响应应触发卡片内容层级切换（Mini / Compact / Full）。

### 2.2 卡片封装组件（CardWrapper.vue）

* 根据卡片实际尺寸决定内容层级展示；
* 使用 `<component :is="renderModes[mode]" />` 动态渲染子组件；
* 每张卡片应带有右上角删除按钮，支持 `emit('remove', id)`；
* 内部通过 ResizeObserver 或 layout.w/h 实现层级判定。

### 2.3 卡片内容层级定义（renderModes）

每种卡片按尺寸区分 3 层级展示：

| 层级      | 用途         | 展示策略            |
| ------- | ---------- | --------------- |
| Mini    | 快速扫读主指标    | 图标 + 数字 + 状态色块  |
| Compact | 展示趋势/摘要    | 简易图表 + 累计值 +小图示 |
| Full    | 展示结构/维度/详情 | 完整图表 + 表格 + 说明  |

> 每张卡片组件应通过 `renderModes = { mini, compact, full }` 显式注册。

## 🔲 2.4 侧边栏卡片组件池（更新版）

侧边栏作为看板的「卡片添加入口」，需同时具备功能性、交互性与 Apple 风格视觉体验。应设计为**可收起的浮动模块**，位于画布左侧，具备以下功能：

### 🎛️ 功能需求

| 功能项         | 描述                              |
| ----------- | ------------------------------- |
| 卡片类型列表      | 显示所有可添加的卡片类型（图标 + 名称）           |
| 拖拽添加        | 支持将卡片拖入主画布，实现拖拽位置即布局落点          |
| 点击添加（快捷添加）  | 单击某卡片类型，则自动添加到默认位置（如画布中心）       |
| 拖拽占位符       | 拖动过程中在画布上预留位置预览（`droppingItem`） |
| 拖拽到被遮挡区域支持  | 即使侧边栏被卡片或其他元素覆盖，依然可完成添加         |
| 卡片默认尺寸规则    | 每个卡片定义初始尺寸，如 `{w: 3, h: 2}`     |
| 拖拽多次稳定性     | 重复拖拽相同类型卡片不会错乱或重叠               |
| 拖拽完成后状态更新顺序 | 需先更新 layout，再添加卡片，避免尺寸失效        |

### ✨ Apple 风格视觉规范

| 元素    | 样式要求                                                    |
| ----- | ------------------------------------------------------- |
| 背景    | 毛玻璃效果 + 半透明 (`backdrop-blur-md` + `bg-white/50`)        |
| 动画    | 淡入淡出动画 + 拖拽悬浮态（如`transition-opacity`、`hover:scale-105`） |
| 边界    | 12px 圆角（`rounded-[12px]`）                               |
| 阴影    | `shadow-lg` + 内部 hover glow                             |
| 拖拽图标  | 每个卡片类型使用统一风格图标组件（SVG 优先）                                |
| 拖拽时行为 | 侧边栏自动进入半透明状态，减少视觉干扰                                     |



### 2.5 主题切换

* 提供 3 套主题：简洁（默认）、深色、科技蓝；
* 使用 Tailwind CSS + class 切换方式；
* 所有组件需响应主题变化；
* 后续可开放用户自定义主题色与字体。

### 2.6 底部信息滚动栏

* 固定在页面底部，用于播放公告、提示、用户自定义文字；
* 支持滚动文字 + 暂停/播放控制；
* 输入历史可持久化保存；
* 后续支持 Markdown/富文本渲染。

## 🧩 三、数据与接口逻辑

### 3.1 数据接口结构

每张卡片需注册：

```ts
{
  id: 'cabinet-status',
  name: '工具柜状态',
  api: '/api/cabinet/status',
  defaultSize: { w: 3, h: 2 },
  renderModes: {
    mini: MiniCard,
    compact: CompactCard,
    full: FullCard
  }
}
```

* `api` 用于数据轮询；
* 系统可设定统一刷新间隔（默认 30 秒）；
* 数据接口返回内容应结构清晰，便于前端映射；
* 每个卡片组件仅关心自身数据，不耦合其它业务。

## 🧠 四、状态管理建议（Pinia）

### 4.1 layoutStore.ts

* 管理卡片的布局信息（x, y, w, h）；
* 提供添加/删除/更新 layout 操作；
* 用于 vue-grid-layout 的响应式绑定。

### 4.2 cardStore.ts

* 存储当前激活的卡片列表；
* 每个卡片的配置、接口、模式等；
* 支持缓存用户添加的卡片顺序与数据。

### 4.3 themeStore.ts

* 当前主题配色（light / dark / custom）；
* 用于样式切换与状态同步；
* 可支持局部响应式主题。

## 🎨 五、UI 风格指南（Apple 风）

* 使用 SF Pro Display / Inter 字体；
* 所有卡片使用浅色底 + 圆角 + 阴影；
* 使用透明度、毛玻璃、柔和边界；
* 图表/图标需克制简洁，不使用过度拟物；
* 组件交互需有过渡/反馈动画（如拖拽、悬停、选中）；
* 保持留白与视觉节奏，控制信息密度。
